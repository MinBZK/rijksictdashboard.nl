Chart layout

Configuration for the layout of the Smart Graph specific for the charts.

Type declaration

  • Optional transitionDuration?: number

    Duration of the transition on redraw.

  • Optional chartMargins?: { top: number; right: number; bottom: number; left: number }

    Margins for the chart to the chart wrapper.

    • top: number
    • right: number
    • bottom: number
    • left: number
  • Optional colors?: string[]

    Colors of the chart.

  • Optional numberScale?: boolean

    Is 'true' when number should be displayed with a 'K', 'M' or 'B' for thousands, millions and billions respectively. For example 1000000 will be 1M . Otherwise 'false'.

  • Optional numberFormat?: string

    Sets the format used for displaying the numbers.

  • Optional showGroupInPopup?: boolean

    Is 'true' when the key/dimension of a group should be shown in the title on hover, otherwise 'false'.

  • Optional rotate?: number

    Number for the rotation of the x axis values for better reading and to lower the bottom margin. Also set this in the css, because otherwise the ticks flicker.

  • Optional textAnchor?: string

    Text-anchor for the x Axis. Use values: "start", "middle", "end"

  • Optional numberOfTicks?: number

    A hint for the number of ticks on the y-axis.

Bar Chart Options

Options for the bar chart.

Type declaration

  • Optional clickableTicks?: boolean

    Is 'true' when clicking on a tick should apply a filter on the bar chart, otherwise 'false'.

  • Optional showInlineValues?: boolean

    Is 'true' when the value of the bars should be shown inside the bar, otherwise 'false'

  • Optional domain?: string[]

    Values displayed on the x-axis of the bar chart. Values not in the domain, are added at the end. Usefull to change the order of the values.

  • Optional type?: "grouped" | "stacked"

    Type of bar chart. Only used for bar chart with multiple dimensions. On 'stacked' groups with the same first dimension will be on top of each other in a color based on the second dimension. On 'grouped' the bars will be next to each other in a color based on the second dimension.

  • Optional showXLabel?: boolean

    Is 'true' when the x label of the bars should be shown inside the bar, otherwise 'false'

  • Optional xLabel?: string

    Value showed as x label.

  • Optional showYLabel?: boolean

    Is 'true' when the y label of the bars should be shown inside the bar, otherwise 'false'

  • Optional yLabel?: string

    Value showed as y label.

  • Optional yLabelPadding?: number

    Padding from the left border of the chart contaienr. Default value: 12

  • Optional gap?: number

    The gap in pixels between the bars. Default value: 2

  • Optional showGridLines?: boolean

    Is 'true' when the gridlines should be shown, otherwise 'false' Default value: false

  • Optional ordering?: "numerical"

    If 'numerical order the bars based on the value' Default value: null

Time Line Options

Options for the time line.

Type declaration

  • Optional clickableTicks?: boolean

    Is 'true' when clicking on a tick should apply a filter on the time line, otherwise 'false'.

  • Optional timeInterval?: "month" | "day" | "week" | "hour" | "minute"

    The interval of time for grouping the dimension values and for filtering the chart.

  • Optional timeFormat?: string

    Format for converting the date time. Usefull when a part of the date time should be ignored, for example if you want to compare two years.

  • Optional showXLabel?: boolean

    Is 'true' when the x label of the bars should be shown inside the bar, otherwise 'false'

  • Optional xLabel?: string

    Value showed as x label.

  • Optional showYLabel?: boolean

    Is 'true' when the y label of the bars should be shown inside the bar, otherwise 'false'

  • Optional yLabel?: string

    Value showed as y label.

  • Optional yLabelPadding?: number

    Padding from the left border of the chart contaienr. Default value: 12

  • Optional showGridLines?: boolean

    Is 'true' when the gridlines should be shown, otherwise 'false' Default value: false

Pie Chart Options

Options for the pie chart.

Type declaration

  • Optional padding?: number

    External padding of the pie chart.

  • Optional externalLabels?: boolean

    Is 'true' when labels should be displayed outside the slices of the pie chart, otherwise false.

  • Optional showAllLabels?: boolean

    Is 'true' when all labels should be displayed without considering the angle of the slice, otherwise false. If true, labels can be hard to read because they overlap.

  • Optional showLabels?: boolean

    Is 'true' when labels should be displayed, otherwise false.

  • Optional legend?: boolean

    Is 'true' when a legend should be added to the pie chart.

  • Optional domain?: string[]

    The order of the values in the pie chart. Values not in the domain, are added at the beginning.

  • Optional ordering?: "numerical"

    If 'numerical order the bars based on the value' Default value: null
